1. Shipped Since the Last Report NEW

Daily Attendance — end to end NEW

  • Class teachers mark attendance one-by-one or roll-call style with a smooth card UI (animated, fullscreen-able)
  • Daily cut-off cron — once the window closes, only the school office can edit, with a full audit trail
  • Parents are notified automatically when their child is marked Absent or Late
  • Monthly heat-map reports per class + per student; "who marked it" report for every entry
  • Leave applications + approval workflow (parent submits → class teacher / admin approves)

Fees + Online Payment (SSLCommerz) NEW

  • School admin can create any type of fee (tuition, exam, transport, lab, late penalty, etc.) with their own labels and amounts
  • Assign a fee to an entire class or to a specific section — every matching student automatically gets an invoice
  • Students / parents see their own invoices in the portal and pay online via SSLCommerz
  • Discounts, scholarships, installments, penalties and refunds all supported on the same fee structure
  • Receipts emailed automatically; 63 automated tests guarding the money flow

Parent + Student Portals NEW

  • Central parent portal at parent-*.doer.school — one login covers all of a parent's children across any Doer school
  • Per-campus student portal at my-*.doer.school — students sign in with their printed Student ID + initial password
  • Forced password change on first sign-in; only students of that campus can log in (no cross-school access)
  • Same Next.js app serves both — copy and navigation switch automatically by portal type
  • Real names + IDs shown (no leaked email handles) — UX polish landed today

Two-Factor Authentication for Doer Staff NEW

  • Authenticator-app (Google Authenticator, Authy, Microsoft Authenticator) enrolment via QR code
  • SMS one-time code fallback for staff without an authenticator app
  • One-time recovery codes printed at setup, regeneratable from the security page
  • "Trust this browser" cookies — staff aren't prompted on every sign-in from their work laptop
  • Production refuses to start with default secrets — hard guard against accidental insecure deployments

Bulk Admission Dry-Run NEW

  • 14 sections seeded in the dev environment — 490 student records modelled on a real campus's roster to mimic production load
  • Student IDs auto-generated from the school's EIIN + section code (e.g. 108357260001)
  • Login accounts auto-provisioned for every imported student, with a per-student initial password
  • Cascading dropdowns + CSV preview catch bad rows before commit
  • "Permanent delete" saga walks all 3 databases atomically — no orphan rows

Campus Config Panel + Section Model v2 NEW

  • Replaces the upfront "pick everything" wizard with on-demand, opt-in configuration per campus
  • Shifts × Sections matrix — one screen to see and edit every class and shift the campus runs
  • Sections now scoped by class × group × shift (matches how real Bangladeshi schools group students)
  • Drag-and-drop subject + book assignment board (one teacher can plan a whole grade's curriculum in minutes)
  • Audit trail on every change (who / when / what)

Teacher Assignment + Home Campus NEW

  • School admin can pick which campus a teacher belongs to (multi-campus schools)
  • Setting an end-date revokes the teacher's login automatically
  • Email notifications on assign / reassign / remove
  • Teacher pool shows name, email, phone, specialisation — recruiters' view

Platform Hardening NEW

  • Unified database migration runner across all 12 services (one command, one rollback)
  • Destructive-migration guard — accidental "drop table" is blocked at the runner
  • System-admin URL rejects tenant logins — staff can't sign in via a parent's URL or vice versa
  • 401 handling overhaul — a missing endpoint no longer kicks the user back to the login screen
  • Active-sessions admin view per role (who's signed in right now)
